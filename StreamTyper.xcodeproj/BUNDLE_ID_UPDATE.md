## 🔍 Bundle ID 快速理解

### 什么是 Bundle ID？
**类比：**
- 就像您的**身份证号** - 唯一标识您
- 就像**车牌号** - 唯一标识您的车
- 就像**域名** - 唯一标识您的网站

**在应用中：**
```
Bundle ID = 应用在 App Store 中的唯一身份证
```

### 为什么重要？
1. **唯一性** - 全世界只有您的应用可以用这个 ID
2. **识别** - 系统用它识别和管理应用
3. **更新** - 更新应用时通过 ID 识别
4. **数据** - 应用数据与 ID 绑定
5. **签名** - 开发者证书与 ID 绑定

### 格式规则
```
✅ 正确: fun.jbzy.TypeStream
❌ 错误: TypeStream
❌ 错误: jbzy.fun.TypeStream (没反向)
❌ 错误: fun.jbzy.Type Stream (有空格)
```

## ❓ 常见问题

### Q: 为什么用反向域名？
**A:** 这是 Apple 和 Java 的传统，确保全球唯一性。
```
您的域名: jbzy.fun
反向后: fun.jbzy
这样可以确保不会与其他公司的应用冲突
```

### Q: .fun 域名可以用吗？
**A:** 完全可以！Apple 支持所有合法域名。
```
✓ .fun
✓ .com
✓ .net
✓ .io
✓ .app
✓ 任何合法域名
```

### Q: 一定要拥有这个域名吗？
**A:** 建议但不强制。
```
推荐（有域名）:
fun.jbzy.TypeStream ← 专业

可以（没域名）:
com.yourname.TypeStream ← 可用但不够正式
```

### Q: 可以有多个应用用同一个域名吗？
**A:** 可以！只要应用名称部分不同。
```
fun.jbzy.TypeStream     ← 剪贴板工具
fun.jbzy.ClipManager    ← 另一个应用
fun.jbzy.QuickType      ← 又一个应用
```

### Q: Bundle ID 区分大小写吗？
**A:** 是的，但建议保持一致。
```
fun.jbzy.TypeStream  ← 推荐（驼峰命名）
fun.jbzy.typestream  ← 也可以（全小写）
fun.jbzy.TYPESTREAM  ← 不推荐（全大写）

选一种风格并始终使用！
```
